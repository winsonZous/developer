### InnoDB与MyISAM存储引擎的区别

InnoDB 支持事务
MyISAM 不支持事务
在同一个事务中，部分表的存储引擎是MyISAM时，这些表的回滚会失败

### 数据库事务的ACID四个特征
数据库主要存放程序运行过程中的数据。
读写数据的过程，如果不加事务控制，是会出问题的，比如转账的例子。
为了解决这些问题，数据库厂商提供数据库的事务控制。包含读写操作的事务必须满足ACID的特性，事务才是正确可靠的。

A:原子性 事务不可分割
C:一致性 总是从一个状态到另外的状态
I:隔离性 两个事务操作数据的可见性
1. 读未提交 脏读
2. 读已提交 不可重复读
3. 可重复读 幻读(MVCC)
4. 串行化 
   D:持久性 事务日志持久化数据到数据文件
